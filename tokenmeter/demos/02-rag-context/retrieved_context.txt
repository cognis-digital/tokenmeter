=== RETRIEVED CHUNK 1 (score 0.91) — docs/install.md ===
Tokenmeter installs from PyPI with `pip install cognis-tokenmeter`. Python 3.10
or newer is required. The CLI entrypoint is `tokenmeter`. On Windows, install
inside a virtual environment to avoid PATH conflicts with system Python.

=== RETRIEVED CHUNK 2 (score 0.88) — docs/cli.md ===
The `count` subcommand reports input tokens, output tokens, and the per-call
cost for a given pricing model. Use `-f` to read a file, `-t` for literal text,
or pipe text on stdin. The `--output-tokens` flag lets you model the expected
completion length so the cost estimate includes generation, not just the prompt.

=== RETRIEVED CHUNK 3 (score 0.84) — docs/budget.md ===
The `budget` subcommand exits with code 1 when an estimate exceeds either a
cost ceiling (`--max-cost`) or a token ceiling (`--max-tokens`), making it
suitable as a CI gate. It also fails when the prompt would exceed the model's
context window, regardless of the configured ceilings.

=== RETRIEVED CHUNK 4 (score 0.79) — docs/models.md ===
Pricing is stored as USD per 1,000 input tokens and per 1,000 output tokens,
together with the model's context window. Run `tokenmeter models` to list every
known model. Prices are data, not code, so they are trivial to update as
vendors change their list prices.
